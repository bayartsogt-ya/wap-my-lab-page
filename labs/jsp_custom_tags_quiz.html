  1. Rewrite the JSP page in the first demo (forEach) to use scripting instead of the JSTL forEach.
    Answer: 
                <%
                for (Student student: (Student[]) request.getAttribute("students")) {
                    out.write("<%tr>");
                    out.write("<%td>" + student.age + "<%/td>");
                    out.write("<%td>" + student.name + "<%/td>");

  2. How do custom tags relate to JSTL?
    Answer:
    JSTL and its 5 different tags are actually custom tags. Basically it provides simpler dynamic html pages.
  3. What is the role of the URI in the TLD and the taglib directive?
    Answer:
    • uri -> source of the code working behind the tag logic (.tld file)
    • tablib directive -> is a way of declaring that using this prefix we should use this custom tag
  4. What is a tag handler class?
    Answer:
    Tag handler is a class extended from SimpleTagSupport and it will provide helper methods which will later be used in JSP file.
  5. What is the role of attribute setters in a tag handler class?
    Answer:
    setters are for getting the inputs from JSP then change the instance variable values.
  6. What is the role of the doTag() method in a tag handler class?
    Answer:
    it is passing a dynamic html element to JSP file using JSPWriter.write
  7. What does the operation getJspContext().getOut().write(“Hi Bob”) do when called in a doTag() method?
    Answer:
    It will add "Hi Bob" simple text element to JSP file.
  8. What does the operation getJspBody().invoke(null) do when called in a doTag() method?
    Answer:
    It will pass the content changed in doTag() method to the client directly instead of the JSP code.
