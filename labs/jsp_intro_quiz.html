JSP Introduction Quiz

Quiz pdf link
  1. What is the main value of Java Server Pages?
    Answer: Simpler interface for developers.
  2. How are JSP pages related to servlets?
    Answer: Making use of the compiler, it can be converted into servlet code.
  3. How are JSP pages related to HTML?
    Answer: After compiling into servlet code (more specifically `out.print`), it can generate dynamic html.
  4. Give an example of a JSP scriptlet and show how it will look in the JSP servlet .
    Answer: <% String username = request.getParameter("username"); %> => String username = request.getParameter("username");
  5. Give an example of a JSP declaration and show how it will look in the JSP servlet.
    Answer:
    <%! String count = 2; %> => String count = 2;
  6. Give an example of a JSP comment and show how it will look in the JSP servlet.
    Answer:
    it will be ignored on "Translation Phase" and will not appear in JSP servlet
    <%-- comment --%> => _nothing_
  7. Give an example of a JSP expression and show how it will look in the JSP servlet.
    Answer:
    <%= count %> => out.print(count)
  8. Give an example of a JSP directive and show how it will look in the JSP servlet.
    Answer:
    <%@page import="java.util.Date" %> => (at the top of JSP servlet) import java.util.Date
  9. Explain how an EL expression is evaluated.
    Answer:
    EL simplifies the JSP expressions.
    ${something}
    1. search in "page scope" or inside the jsp
    2. otherwise search in "request scope"
    3. otherwise search in "session scope"
    4. otherwise search in "application scope" such as among the models
    5. otherwise ignore
  10. Explain how servlet attributes are involved in EL expressions.
    Answer:
    If it is asking about the servlet as a controller dispatching into JSP file, EL cannot find an involvement there. However, if it is asking servlet as a compiled JSP file, it can be any match of attributes written in JSP expressions.
  11. Explain how servlets and JSPs use request dispatch to interact
    Answer:
    Request Dispatcher in Servlet (controller) directly calling the JSP file (view) from file system to create dynamic html and send it for the incoming request as a response.
